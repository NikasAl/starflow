## 1. Обзор

### 1.1 Концепция

**Командование потоками** — 3D-стратегия в реальном времени, в которой игрок управляет маршрутами ракет между планетами. Каждая планета имеет один числовой показатель — мощность. Ракеты летят по постоянным маршрутам и при попадании в планету изменяют её мощность: свои ракеты увеличивают, чужие — уменьшают. При падении мощности до нуля планета захватывается атакующим.

Задача — перекрасить все планеты в свой цвет, создавая сеть маршрутов, которая обеспечивает постоянное давление на вражеские позиции и защитные подкрепления собственных планет. Ключевой стратегический вызов — баланс между атакой и обороной: отправленные на атаку ракеты ослабляют источник, а слишком пассивная оборона позволяет противнику захватить инициативу.

### 1.2 Жанр и целевая аудитория

**Жанр:** 3D-стратегия реального времени с упором на маршрутизацию и макро-управление.

**Целевая аудитория:** игроки, предпочитающие стратегическое планирование и логистику. Игровой процесс не требует микроменеджмента отдельных юнитов — всё строится на создании и разрыве маршрутов. Оптимальная продолжительность партии: 3–10 минут. Платформы: веб (браузер), Android (мобильные устройства). Управление мышью (ПК) и тач (мобильные).

### 1.3 Уникальные преимущества

- **Система единой мощности:** интуитивная и прозрачная модель — одна цифра на планете, никаких сложных таблиц юнитов и ресурсов. Всё, что нужно знать, видно сразу.
- **Маршрутная логистика:** игрок создаёт постоянные каналы снабжения и атаки, а не отправляет отдельные отряды. Это ближе к управлению логистической сетью, чем к классическим RTS.
- **Динамические лимиты:** количество маршрутов от планеты зависит от мощности, создавая «снежный ком» — подкреплённая планета получает больше маршрутов и может атаковать по более широкому фронту.
- **Процедурные текстуры:** каждая планета получает уникальную текстуру с учётом типа и цвета владельца, создавая визуально разнообразную карту без внешних активов.
- **Простое управление:** клик-клик для создания маршрутов, без перетаскиваний, контекстных меню и сложных комбинаций.

## 2. Основные механики

### 2.1 Система планет и мощность

Планета — основной объект игрового мира. Каждая планета определяется следующими свойствами:

| Параметр | Тип | Описание |
|----------|-----|----------|
| `position` | `{x, y, z}` | Позиция в 3D-пространстве |
| `radius` | `float` | Визуальный радиус сферы (зависит от тира) |
| `owner` | `OwnerId` | Владелец: нейтрал, игрок, ИИ-1, ИИ-2 |
| `power` | `number` | Текущая мощность (основной показатель) |
| `tier` | `1 \| 2 \| 3` | Размер: малая, средняя, крупная |
| `visualType` | `string` | Тип текстуры (rocky, terran, gas, ice и т.д.) |

Мощность — единый показатель, заменяющий все отдельные ресурсы (истребители, крейсеры, гарнизон). Она одновременно является:
- **Защитой:** чем выше мощность, тем сложнее захватить планету
- **Производственным потенциалом:** напрямую не производит ракет, но определяет лимит маршрутов
- **Показателем стратегической ценности:** мощная планета может атаковать по нескольким направлениям

#### Тиры планет

Планеты трёх размеров генерируются процедурно с вероятностями:

| Тир | Вероятность | Радиус | Сила ракеты | Описание |
|-----|------------|--------|-------------|----------|
| 1 | 50% | 1.2 | 1 | Малые — быстрые, но слабые ракеты |
| 2 | 35% | 1.8 | 1 | Средние — основной тип |
| 3 | 15% | 2.5 | **2** | Крупные — ракеты наносят двойной урон |

Крупные планеты (тир 3) стратегически ценны: их ракеты меняют мощность цели на 2 единицы вместо 1, что удваивает скорость захвата. Однако это компенсируется редкостью — только 15% планет на карте.

### 2.2 Ракеты

Ракета — единственный боевой юнит. Представляет собой снаряд, летящий от планеты-источника к планете-цели по прямой линии.

| Характеристика | Значение | Настройка |
|---------------|----------|-----------|
| Скорость | 20 ед./сек | `MISSILE_SPEED` |
| Траектория | Прямая линия | — |
| Сила (тир 1, 2) | 1 | `getMissileStrength(tier)` |
| Сила (тир 3) | 2 | `getMissileStrength(tier)` |
| Интервал отправки | 3 сек/маршрут | `ROUTE_SEND_INTERVAL` |

Ракеты летят **по одной**, последовательно. Каждая ракета — самостоятельный объект с собственной позицией, прогрессом полёта и владельцем. При прибытии к цели ракета взаимодействует с планетой и удаляется.

Визуально ракета изображается **цилиндром** с носовым конусом (полусфера) и свечением двигателя (светящаяся полусфера в корме). Цилиндр ориентирован по вектору движения в мировых координатах. Цвет ракеты соответствует цвету владельца-отправителя.

### 2.3 Система маршрутов

Маршрут — постоянная связь между двумя планетами, по которой автоматически отправляются ракеты с фиксированной частотой.

| Свойство маршрута | Описание |
|------------------|----------|
| `sourceId` | ID планеты-источника |
| `targetId` | ID планеты-цели |
| `owner` | Владелец маршрута |
| `missileStrength` | Сила ракеты (1 или 2, зависит от тира источника) |
| `sendTimer` | Таймер обратного отсчёта до следующей ракеты |

#### Лимит маршрутов от планеты

Количество исходящих маршрутов от одной планеты ограничено её текущей мощностью:

| Мощность | Макс. маршрутов | Функция |
|----------|----------------|---------|
| 1 — 15 | 1 | `getMaxRoutesFromPlanet(15)` |
| 16 — 30 | 2 | `getMaxRoutesFromPlanet(30)` |
| 31+ | 3 | `getMaxRoutesFromPlanet(31)` |

Это ключевая механика, создающая экономическую систему:
- Новая планета (мощность 5-10) может атаковать только по **одному** направлению
- Подкреплённая планета (мощность 20+) может атаковать по **двум** направлениям
- Мощная планета (мощность 35+) атакует по **трём** направлениям

Это порождает стратегический цикл: захват → подкрепление → расширение фронтов → захват новых планет.

#### Ограничения маршрутов
- Запрещены маршруты на саму себя
- От одной планеты к другой — только один маршрут (toggle при повторном клике)
- Маршрут автоматически удаляется, если источник потерян (захвачен противником)
- Маршрут не создаётся, если мощность источника < 1

### 2.4 Боевая механика

Взаимодействие ракеты с планетой определяется сравнением цвета ракеты и цвета владельца планеты.

#### Подкрепление (свой цвет)

Если ракета того же цвета, что и владелец планеты:
- `power = power + missileStrength`
- Мощность увеличивается без ограничений

Это позволяет усиливать планеты для увеличения лимита маршрутов.

#### Атака (чужой цвет)

Если ракета другого цвета:
- `power = power - missileStrength`
- Если `power > 0` после вычитания — планета держится, новый владелец не устанавливается
- Если `power <= 0` — планета **захватывается** атакующим, и остаток мощности переносится:
  - `newPower = abs(power)` (например: power=3, missile=-5 → захват, newPower=2)

#### Примеры расчётов

| Сцена | До | Ракета | Результат |
|-------|-----|--------|-----------|
| Подкрепление малое | 12 | +1 | **13** (владелец тот же) |
| Подкрепление мощное | 12 | +2 | **14** (тир 3) |
| Лёгкая атака | 8 | -1 | **7** (защита держится) |
| Мощная атака | 8 | -2 | **6** (защита держится) |
| Захват точно | 2 | -2 | **0** (захват, newPower=0) |
| Захват с остатком | 5 | -8 (4 ракеты) | **3** для атакующего |
| Перекрытие | 1 | -2 | **1** для атакующего |

### 2.5 Автоматический рост мощности

Все захваченные планеты (игроком или ИИ) автоматически наращивают мощность:

- **Скорость роста:** 1 единица в секунду (`PLANET_POWER_GROWTH_RATE`)
- **Порог:** только до 10 (`PLANET_MAX_AUTO_POWER`)
- Нейтральные планеты **не растут**

Мощность выше 10 достижима **только через подкрепление** ракетами. Это создаёт чёткую границу: базовая защита 10 обеспечивается автоматически, но для offensive/defensive расширения нужны активные действия игрока.

**Стратегическое значение:**
- Изолированная планета всегда имеет минимум 10 единиц защиты через 10 секунд после захвата
- Планета с мощностью 15+ может поддерживать 1 маршрут, но без подкреплений не перейдёт к 2 маршрутам
- Подкрепление «своих» планет — ключ к расширению фронта атак

## 3. Игровой ввод

### 3.1 Управление камерой

Орбитальная камера с центром вращения в начале координат.

| Действие | Управление |
|----------|-----------|
| Вращение | Перетаскивание мышью (ЛКМ) |
| Панорамирование | Shift + перетаскивание |
| Масштаб | Колёсико мыши |
| Предел приближения | 20 — 150 единиц |
| Угол наклона | Ограничен (0.2 — 1.37 рад, ~12° — ~79°) |

Камера конфигурируется константами `CAM_DEFAULT_DISTANCE`, `CAM_MIN_DISTANCE`, `CAM_MAX_DISTANCE`, `CAM_ZOOM_SPEED`.

### 3.2 Создание маршрутов

Ввод строится на двух кликах (click-click), без перетаскиваний.

1. **Клик по своей планете** — выделение (появляется зелёное пульсирующее кольцо, HUD показывает детали)
2. **Клик по цели** — создание маршрута (появляется пульсирующая линия связи)
3. **Клик по источнику повторно** — удаление всех маршрутов от этой планеты
4. **Клик по пустому пространству** — снятие выделения

Если кликнуть по цели, к которой уже ведёт маршрут от выделенного источника — маршрут удалится (toggle).

**Ограничения:**
- Можно выбирать только свои планеты
- При исчерпании лимита маршрутов показывается ошибка в HUD
- Нельзя создавать маршруты с мощностью источника < 1

## 4. Условия победы и поражения

| Состояние | Условие | Результат |
|----------|---------|-----------|
| **Победа** | Игрок владеет всеми 12 планетами | Экран VICTORY |
| **Поражение** | Игрок не владеет ни одной планетой | Экран DEFEAT |
| **Игра** | Все три фракции имеют хотя бы по 1 планете | Продолжается |

Игра заканчивается мгновенно при выполнении условия — нет таймеров, дополнительного времени или последних шансов. Это создаёт напряжение в моменты, когда одна из фракций теряет предпоследнюю планету.

## 5. ИИ-противники

### 5.1 Алгоритм принятия решений

ИИ работает циклически с интервалом ~2 секунды (с рандомизацией для избежания синхронности двух ИИ).

Каждый цикл состоит из трёх фаз:

**Фаза 1 — Очистка:**
- Удаление маршрутов от потерянных планет (захвачены противником)
- Удаление маршрутов к собственным планетам с мощностью > 12 (достаточно укреплены, освобождаем слоты)

**Фаза 2 — Расширение:**
- Сортировка своих планет по мощности (убывание)
- Для каждой планеты: проверка свободных слотов маршрутов
- При наличии свободного слота — поиск лучшей цели

**Фаза 3 — Выбор цели:**
- Расчёт оценки для каждой доступной цели:
  - Вражеская/нейтральная: `score = distance + power * 1.5`
  - Своя (подкрепление): `score = distance * 1.5 + power * 3` (меньший приоритет)
- Выбор цели с минимальной оценкой
- Создание маршрута с силой ракеты по тиру источника

### 5.2 Фракции

| Фракция | Идентификатор | Цвет | Поведение |
|---------|--------------|------|-----------|
| **Player** | `PLAYER (1)` | Синий (0x4488ff) | Управляется игроком |
| **Crimson Fleet** | `AI_1 (2)` | Красный (0xff4444) | Агрессивная экспансия |
| **Emerald Horde** | `AI_2 (3)` | Зелёный (0x44cc44) | Агрессивная экспансия |

ИИ не различается по профилям — оба используют одинаковый алгоритм с естественной вариацией из-за рандомизации таймеров и разных стартовых позиций. Различное поведение возникает из контекста: ближние к слабым нейтральным планетам ИИ захватывают их быстрее, создавая асимметричную динамику.

## 6. Визуальная система

### 6.1 Планеты

Планеты визуализируются как **текстурированные сферы** (SphereGeometry, 48 сегментов × 32 кольца) с PBR материалом (MeshStandardMaterial).

**Текстурирование:**
- Процедурная генерация через Canvas 2D API (512×256 пикселей)
- 8 типов планет: rocky, terran, gas, ice, volcanic, desert, ocean, crystal
- Каждая текстура включает diffuse-карту, normal-карту и (для volcanic/crystal) emissive-карту
- Генерация основана на шуме (value noise, FBM, warped noise) с сидом для детерминизма
- Текстуры тонированы в цвет владельца на 40% (ownerTint) для наглядности

**Принадлежность:**
- Emissive-цвет материала = цвет владельца (intensity 0.2)
- Кольцо-аура под планетой = цвет владельца (opacity 0.4)
- Метка (Sprite) над планетой: мощность крупно + макс. маршрутов

**Планеты медленно вращаются** (0.003 рад/кадр) для визуального эффекта.

### 6.2 Ракеты

Ракеты визуализируются как **группы из трёх мешей:**
- **Цилиндр** (CylinderGeometry, 0.12 радиус × 0.8 высота) — корпус
- **Носовой конус** (SphereGeometry, 0.15 радиус) — в передней части
- **Свечение двигателя** (SphereGeometry, 0.2 радиус, полупрозрачное) — в корме

Цилиндр ориентирован по вектору движения через кватернион. Цвет — цвет владельца с emissive свечением (intensity 0.6).

### 6.3 Маршруты

Маршруты отображаются **пульсирующими линиями** (LineBasicMaterial) между позициями планет-источников и целей.
- Цвет = цвет владельца маршрута
- Opacity осциллирует: `0.2 + sin(time * 2 + id.length) * 0.15`
- Линии создаются и уничтожаются при создании/удалении маршрутов

### 6.4 HUD

HUD реализован как **HTML overlay** (position: fixed), не зависящий от 3D камеры.

Структура панели (верхний левый угол):
- **Таймер:** MM:SS
- **Статистика фракций:** цветной индикатор, название, кол-во планет, суммарная мощность
- **Маршруты игрока:** счётчик активных маршрутов
- **Ракеты:** счётчик ракет в полёте
- **Выбранная планета:** имя, мощность, текущие/макс. маршруты
- **Состояние игры:** VICTORY (зелёный) или DEFEAT (красный)

Фон: rgba(0,0,0,0.7) с backdrop-filter: blur(4px). Шрифт: Segoe UI / Arial.

### 6.5 Звёздное поле

Фоновые звёзды — 2000 точек (Points), распределённых сферически на расстоянии 150–350 единиц от центра. Создают ощущение космического пространства при вращении камеры.

### 6.6 Освещение

| Свет | Тип | Интенсивность | Позиция |

## 8. Техническая архитектура

### 8.1 Стек технологий

| Технология | Назначение |
|-----------|-----------|
| **Three.js** | 3D рендеринг (WebGL), сцена, камеры, PBR материалы |
| **TypeScript** | Типобезопасная игровая логика, компиляция ES2020 |
| **Vite** | Dev-сервер (HMR), сборка, tree-shaking |
| **Capacitor 6** | Обёртка для Android APK |
| **Canvas 2D API** | Процедурная генерация текстур планет |

### 8.2 Структура кода

```
starflow-3d/
├── src/
│   ├── core/                    # Чистая игровая логика (без Three.js)
│   │   ├── types.ts             # PlanetData, MissileData, ShipRoute, GameState
│   │   ├── constants.ts         # Баланс, камера, ИИ
│   │   ├── planet.ts            # generateMap(), updatePlanetGrowth(), resolveMissileArrival()
│   │   ├── fleet.ts             # createMissile(), updateMissile()
│   │   ├── ai.ts                # createAIs(), updateAI()
│   │   └── texture-gen.ts       # generatePlanetTextures(), 8 типов, noise/FBM
│   ├── game/
│   │   ├── state.ts             # createGameState(), updateGame(), handlePlayerAction()
│   │   └── game.ts              # startGame(), gameLoop(), синхронизация
│   ├── rendering/
│   │   └── renderer.ts          # Three.js: сцена, планеты, ракеты, маршруты, HUD, камера
│   └── main.ts                  # Точка входа: DOMContentLoaded → startGame()
├── index.html                   # Canvas + инструкции
├── package.json                 # Зависимости и скрипты
├── vite.config.ts               # Конфигурация Vite
├── tsconfig.json                # Конфигурация TypeScript
└── capacitor.config.ts          # Конфигурация Capacitor (Android)
```

### 8.3 Принцип разделения

Архитектура следует принципу **разделения логики и визуализации:**

- **`core/`** — чистые функции и типы. Не импортирует Three.js. Можно тестировать независимо.
  - `types.ts` — определения данных
  - `constants.ts` — параметры баланса
  - `planet.ts` — генерация карты, рост мощности, разрешение боя (чистые функции)
  - `fleet.ts` — создание и обновление ракет (чистые функции)
  - `ai.ts` — алгоритм ИИ (чистая функция updateAI, принимает состояние, возвращает изменения)
  - `texture-gen.ts` — генерация текстур через Canvas 2D (зависит от Three.js для CanvasTexture, но генерация пикселей чистая)

- **`game/`** — оркестрация. Импортирует core и координирует обновление состояния.
  - `state.ts` — центральный game loop: производство → ИИ → маршруты → ракеты → прибытия → победа
  - `game.ts` — связывает game state с renderer, отслеживает новые/удалённые объекты

- **`rendering/`** — визуализация. Импортирует Three.js и game state. Только читает данные.
  - `renderer.ts` — создаёт и обновляет все Three.js объекты, HUD, обрабатывает ввод

## 9. Сборка и развёртывание

### 9.1 Веб-версия

```bash
npm run dev       # Dev server: http://localhost:3001 (HMR)
npm run build     # TypeScript check → Vite build → dist/
npm run preview   # Preview production build
```

### 9.2 Android

```bash
# Первоначальная настройка (один раз)
npm run cap:init          # Создаёт capacitor.config.ts
npm run cap:add:android   # Создаёт android/ директорию

# Сборка и развёртывание
npm run build             # Собрать веб → dist/
npm run cap:sync          # Скопировать dist/ в android/app/src/main/assets/public/
npm run cap:open:android  # Открыть Android Studio → Build → Run

# CLI-сборка (без Android Studio)
cd android
./gradlew assembleDebug
# APK: android/app/build/outputs/apk/debug/app-debug.apk

# Установка на подключённое устройство
./gradlew installDebug
```

**Требования:** Android Studio или SDK Command-line Tools, Android SDK 33+, Java 17+.
