## Правила игры

### Планеты и мощность

Каждая планета имеет **единственный числовой показатель — мощность (power)**. Мощность определяет всё: защиту, атаку, количество доступных маршрутов. У планеты нет отдельных истребителей, крейсеров или других юнитов — только одна цифра.

Планеты бывают трёх размеров (тиров), которые влияют на визуальный размер и силу отправляемых ракет:

| Тир | Радиус сферы | Сила ракеты | Описание |
|-----|-------------|-------------|----------|
| 1 | 1.2 | 1 | Малая планета |
| 2 | 1.8 | 1 | Средняя планета |
| 3 | 2.5 | **2** | Крупная планета (ракеты мощнее!) |

На старте игры каждый из трёх игроков (Вы, Crimson Fleet, Emerald Horde) получает по одной средней планете с мощностью 15. Остальные 9 планет — нейтральные, с мощностью от 5 до 10.

### Ракеты

Единственный боевой юнит в игре. Ракета — это снаряд, летящий от планеты-источника к планете-цели по **прямой линии**. Визуально изображается цилиндром с носовым конусом и свечением двигателя.

| Характеристика | Значение |
|---------------|----------|
| Скорость полёта | 20 ед./сек |
| Траектория | Прямая линия (без дуги) |
| Сила (тир 1, 2) | 1 единица мощности |
| Сила (тир 3) | 2 единицы мощности |
| Интервал отправки | 1 ракета каждые 3 секунды (на маршрут) |

Ракеты летят **по одной** — нет групп или флотов. Каждая ракета — индивидуальный снаряд, который прилетает и взаимодействует с планетой независимо от других.

### Система маршрутов

Маршрут — это постоянная связь между двумя планетами. После создания маршрута ракеты отправляются **автоматически** с заданной частотой. Игроку не нужно вручную отправлять каждую ракету — достаточно создать маршрут, и ракеты будут лететь непрерывно.

**Создание маршрута:**
1. Кликните по своей планете (выделение — зелёное пульсирующее кольцо)
2. Кликните по планете-цели (любой, в том числе своей)
3. Маршрут создан — видна пульсирующая линия связи

**Удаление маршрута:**
- Кликните по планете-источнику повторно — удалятся все её исходящие маршруты
- Или кликните по уже подключённой цели — удалится конкретный маршрут (toggle)

**Лимит маршрутов от одной планеты** зависит от её текущей мощности:

| Мощность планеты | Макс. исходящих маршрутов |
|-----------------|--------------------------|
| 1 — 15 | **1** маршрут |
| 16 — 30 | **2** маршрута |
| 31 и выше | **3** маршрута |

Если лимит исчерпан, при попытке создать маршрут HUD покажет ошибку: «Max N route(s) for power X». Это ключевая механика — чтобы атаковать по нескольким направлениям, планета должна быть мощной.

### Боевая механика

Когда ракета долетает до планеты-цели, происходит взаимодействие на основе цвета ракеты и цвета владельца планеты:

**Свой цвет (подкрепление):**
- Мощность планеты **увеличивается** на силу ракеты (+1 или +2)
- Полезно для усиления обороны и подготовки к атаке

**Чужой цвет (атака):**
- Мощность планеты **уменьшается** на силу ракеты (-1 или -2)
- Если мощность упала до **0 или ниже** — планета **захватывается** атакующим
- При захвате остаток мощности переносится новому владельцу (модуль отрицательного значения)

**Примеры:**
| Ситуация | Мощность до | Сила ракеты | Результат |
|----------|------------|------------|-----------|
| Подкрепление | 12 | +1 | Мощность = 13 |
| Лёгкая атака | 5 | -1 | Мощность = 4 |
| Мощная атака (тир 3) | 5 | -2 | Мощность = 3 |
| Захват | 3 | -2 | Захват! Мощность = 1 (новый владелец) |
| Точный захват | 2 | -2 | Захват! Мощность = 0 |
| Перекрытие | 1 | -2 | Захват! Мощность = 1 |

### Автоматический рост мощности

Планеты, принадлежащие игроку или ИИ, **самостоятельно наращивают мощность** со скоростью **1 единица в секунду**, но **только до 10**. Это базовый прирост, который гарантирует, что даже изолированная планета не останется совсем без защиты.

Мощность выше 10 достигается **только за счёт входящих ракет** (подкреплений). Это создаёт важный стратегический выбор: отправлять все ракеты в атаку или часть направлять на подкрепление своих планет, чтобы увеличить их лимит маршрутов.

Нейтральные планеты **не растут** — их мощность статична, пока кто-то не захватит их.

### Победа и поражение

- **Победа:** вы захватили все планеты на карте
- **Поражение:** вы потеряли все свои планеты
- Игра продолжается, пока одна из фракций не станет единственным владельцем всех планет

## ИИ-противники

Два ИИ-противника с агрессивной стратегией расширения:

| Фракция | Цвет |
|---------|------|
| **Crimson Fleet** | Красный (0xff4444) |
| **Emerald Horde** | Зелёный (0x44cc44) |

### Алгоритм ИИ

ИИ оценивает ситуацию каждые **2 секунды** (с рандомизацией) и выполняет следующие действия:

1. **Очистка:** удаление маршрутов от потерянных планет и к собственным планетам с мощностью > 12 (уже закреплены)
2. **Расширение:** создание маршрутов со всех планет, где есть свободные слоты
3. **Приоритеты целей:**
   - Вражеские и нейтральные планеты: приоритет = расстояние + мощность * 1.5 (ближайшие слабые — первыми)
   - Свои слабые планеты: приоритет = расстояние * 1.5 + мощность * 3 (меньший приоритет, чем атака)
4. **Сортировка:** планеты с большей мощностью получают маршруты первыми

ИИ активно расширяет сеть, не «зависает» и постоянно перенаправляет маршруты по мере изменения ситуации на карте.

## Визуальные элементы

### Планеты
- **Сферы** с процедурно генерируемыми текстурами (8 типов: rocky, terran, gas, ice, volcanic, desert, ocean, crystal)
- Текстуры **тонированы в цвет владельца** (40% mix) для наглядности принадлежности
- **Свечение (emissive)** в цвет владельца усиливает визуальное различие
- **Кольцо-аура** под планетой — цвет владельца, полупрозрачное
- **Метка** над планетой: мощность (крупно) + макс. маршрутов
- Планеты медленно вращаются

### Ракеты
- **Цилиндры** с носовым конусом и свечением двигателя
- Ориентированы по вектору движения (направлены к цели)
- Цвет — цвет владельца-отправителя
- Крупные ракеты (тир 3) визуально не отличаются, но наносят x2 урон

### Маршруты
- **Пульсирующие линии** между планетами (цвет владельца)
- Интенсивность пульсации варьируется для различения маршрутов

### HUD (экранный интерфейс)
- **Фиксированная панель** в верхнем левом углу (не зависит от камеры)
- Для каждой фракции: цветной индикатор, название, кол-во планет, суммарная мощность
- Счётчик активных маршрутов игрока
- Счётчик ракет в полёте
- При выделении: имя планеты, мощность, текущие/макс. маршруты
- Статус игры: таймер, VICTORY / DEFEAT

### Звёздное поле
- 2000 фоновых звёзд, сферически распределённых
- Создаёт ощущение космического пространства

## Параметры баланса

Все ключевые параметры вынесены в `src/core/constants.ts` и могут быть изменены без переписывания логики:

```typescript
// Размер мира
WORLD_SIZE = 80
PLANET_MIN_DISTANCE = 18
PLANET_COUNT = 12

// Мощность
STARTING_POWER = 15
NEUTRAL_POWER_MIN = 5
NEUTRAL_POWER_MAX = 10
PLANET_MAX_AUTO_POWER = 10       // Авто-рост только до этого значения
PLANET_POWER_GROWTH_RATE = 1.0   // Единиц в секунду

// Маршруты
ROUTE_SEND_INTERVAL = 3.0        // Секунд между ракетами (настраиваемо!)
// getMaxRoutesFromPlanet(power): 15->1, 30->2, 30+->3
// getMissileStrength(tier): 1,2->1, 3->2

// Ракеты
MISSILE_SPEED = 20               // Единиц в секунду

// ИИ
AI_THINK_INTERVAL = 2.0          // Секунд между решениями

// Камера
CAM_DEFAULT_DISTANCE = 60
CAM_MIN_DISTANCE = 20
CAM_MAX_DISTANCE = 150
```

## Android сборка

```bash
# Первоначальная настройка (один раз)
npm run cap:init          # Создаёт capacitor.config.ts
npm run cap:add:android   # Создаёт android/ директорию

# Повторная сборка
npm run build             # Собрать веб -> dist/
npm run cap:sync          # Синхронизировать dist/ в android/
npm run cap:open:android  # Открыть Android Studio

# Или CLI-сборка без Android Studio:
cd android && ./gradlew assembleDebug
# APK: android/app/build/outputs/apk/debug/app-debug.apk
```

**Требования:** Android Studio (или Gradle CLI), Android SDK 33+, Java 17+

## Архитектура

```
src/
├── core/              # Чистая игровая логика (без Three.js)
│   ├── types.ts       # PlanetData, MissileData, ShipRoute, GameState, OwnerId
│   ├── constants.ts   # Баланс, камера, ИИ параметры
│   ├── planet.ts      # generateMap(), updatePlanetGrowth(), resolveMissileArrival()
│   ├── fleet.ts       # createMissile(), updateMissile() — прямая линия
│   ├── ai.ts          # createAIs(), updateAI() — расширение и маршрутизация
│   └── texture-gen.ts # Процедурные текстуры: noise, FBM, 8 типов планет
├── game/
│   ├── state.ts       # createGameState(), updateGame(), handlePlayerAction()
│   └── game.ts        # startGame(), gameLoop(), синхронизация с рендерером
├── rendering/
│   └── renderer.ts    # Three.js: сцена, планеты, ракеты, маршруты, HUD, камера
└── main.ts            # DOMContentLoaded -> startGame()
```

**Принцип разделения:** `core/` не зависит от Three.js — чистые функции и типы. `rendering/` отвечает только за визуализацию. `game/` связывает логику с рендерером. Это позволяет тестировать игровую логику независимо от графики.

## Tech Stack

- **Three.js** — 3D рендеринг (сферы, цилиндры, частицы, PBR материалы, нормал-маппинг)
- **TypeScript** — типобезопасная игровая логика
- **Vite** — быстрый dev-сервер и сборщик
- **Capacitor 6** — обёртка для Android
